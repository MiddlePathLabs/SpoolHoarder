Changelog - Recent Updates (May - July 2026)

## Version 2.5.2
[ Features ]
- Linked NFC tags to individual spools and added an inventory action that scans a linked tag and jumps straight to its spool, without changing the tag's existing data.
- Added an optional mobile setting to write a companion link on NFC tags that opens the spool directly when the system reads the tag; off by default because some OpenSpool readers may not support tags with extra records.
- Expanded cloud sync (Pro accounts) to include NFC tag bindings, so tags set up on one device work on the rest.


## Version 2.5.1
[ Features ]
- Restored on-device barcode scanning on Android, bringing it back to parity with other platforms for fast local scans without needing the cloud.
- Added missing default images for several Bambu Lab filaments, including PLA Pure, PLA Tough, and a few multi-color and TPU colors that previously showed a generic placeholder.

[ Fixes ]
- Resolved subscription redemption and entitlement issues that affected some Android purchases through Google Play.
- Improved camera stability when leaving the scanner screen.
- Improved how filament product images are matched so the correct image appears more reliably.

[ Security & Infrastructure ]
- Reduced the app's download and installed size by trimming oversized bundled images.


## Version 2.5.0
[ Features ]
- Added a dense table view for inventory on desktop — resizable panes, click-to-sort columns, and a docked details inspector — built for managing large collections at a glance.
- Redesigned the Stats overview into an operational report that surfaces readiness and "needs attention" signals more clearly.
- Widened the desktop workspace with a denser layout across inventory, projects, and spool details.
- Added a skeleton loading state so inventory shows its structure while your data loads.
- Added an option when importing usage history from a backup: apply it to reduce your current spool weights, or import history-only and leave weights unchanged.

[ UI & UX ]
- Rebuilt the add-spool intake with a live inspector preview that mirrors the inventory card, including low-stock indication.
- Compacted the spool detail primary controls and aligned the weight and actions panels for a cleaner layout.
- Widened the projects workspace on desktop with a tighter, more informative project record layout.
- Polished spool cards with a dedicated favorite-star color, cleaner selection highlighting, and a simplified weight badge.
- Gave the Calculator tab a distinct selected icon so it no longer clashes with Stats.
- Sharpened inventory interactions with clearer focus feedback, persistent pane widths, and extended desktop keyboard controls.
- Improved dark themes: restored contrast in the Onyx and Copper palettes and normalized status colors.
- Adapted the filament catalog search picker for desktop layouts.
- Surfaced explanations for inventory color advice so you can see why a color is suggested.
- Let the inventory empty-state message wrap naturally on narrow screens.
- Added screen-reader progress announcements across spool save, order and library imports, barcode capture, and code redemption.
- Honored the system reduced-motion preference throughout the app, and preserved touch-target sizes and text scaling on more controls.

[ Fixes ]
- Fixed automatic cloud sync to retry after transient failures, and repaired account ownership for anonymous accounts.
- Made Pro/subscription status appear instantly from a cached state while it revalidates in the background.
- Made sorted spool order deterministic so lists stay stable between refreshes.
- Honored your configured low-stock threshold in stats.
- Scaled the monthly stats chart correctly at large text sizes.
- Resolved rendering, hierarchy, and copy issues found in a full UI review.
- Fixed a bug where the on-device assistant could occasionally return an answer that didn't match the question you asked.
- Fixed misaligned action buttons in the spool detail header, and a clipped weight label on some spool cards.
- Fixed multi-color suggestions in the color advisor not showing their color swatches.
- Fixed a crash that could occur when returning to a spool or project detail screen after the record had changed elsewhere.
- Fixed the filament catalog auto-match: barcode scans no longer match non-product barcodes, and a match in progress is no longer lost if you save the spool before it finishes loading.
- Fixed AI scan credits occasionally being charged twice when retrying a failed or timed-out scan; canceling a scan now properly stops it instead of leaving it running, and error messages are clearer when a scan fails due to a network or access issue.
- Fixed AI order import leaving a stale credit count after running out of credits mid-import.
- Fixed the trial period status sometimes carrying over between different accounts on the same device.
- Fixed community filament measurement contributions not being sent for guest/anonymous accounts, and deleted projects occasionally reappearing after a cloud sync or restore.
- Fixed an interrupted usage-history import potentially leaving spool weights partially updated.

[ Security & Infrastructure ]
- Improved app startup time and reduced unnecessary re-rendering across inventory, spool details, and stats for smoother scrolling.
- Made cloud sync more resilient to network stalls by adding timeouts and automatic retries, so a stuck connection can no longer block subsequent syncs.

## Version 2.4.2
[ Features ]
- Expanded cloud sync (Pro accounts) to cover printers, printer-specific spool settings, project filament assignments, and project pricing.
- Added desktop keyboard shortcuts for inventory: Ctrl/Cmd+F to search, Ctrl/Cmd+N to add a spool, and Esc to close panels.
- Added one-tap removal of onboarding sample data.
- Added support for using your own AI API key for scanning.
- Made stats "Needs Attention" tiles (low stock, missing printer, missing location, missing technical profile, missing dry history) tappable to jump straight to filtered inventory.
- Added a validation summary that lists missing required fields when saving a spool fails, and moves focus to the first missing field.
- Showed spool card display-field summary pills on desktop compact rows and group cards.
- Added automatic filament identification when adding or batch-scanning spools: barcode and photo scans can now match against a filament catalog to suggest brand, material, and color, with a manual search fallback when there's no confident match.

[ UI & UX ]
- Redesigned the compact inventory toolbar with filter/sort as icon buttons showing an active-filter badge, and moved batch Select into the view-options menu.
- Refreshed the first-run empty state with the app's signature liquid-fill tank graphic and a bolder headline.
- Streamlined the upgrade dialog, renamed the inventory batch "Empty" action to "Mark Empty", and fixed a Calculator/Stats tab icon clash.
- Refreshed upgrade/subscription prompts throughout the app with clearer, benefit-focused messaging.
- Replaced the generic sparkle icon with context-specific icons throughout the app (assistant, AI scan, order import, and more).
- Improved assistant results to show rich, familiar spool-card previews inline, with clearer action buttons.
- Reworded onboarding copy to focus on outcomes rather than setup steps.
- Made Retry the primary action on load-error screens, with destructive actions shown as outlined instead of filled.
- Made in-app confirmation and error messages consistent across the app, with reliable screen-reader announcements.
- Clarified spool card badges: temperature chips now read "240°C nozzle"/"90°C bed", and low-stock badges read "Low stock" instead of "Low".
- Showed active filter status on phones while browsing inventory (previously desktop-only), and added a Select None action to batch mode.
- Added color swatches to the palette picker in appearance settings.
- Added a leading back button to the Print Library screen on narrow widths.
- Added accessible labels and larger tap targets to interactive elements throughout the app.
- Made the barcode field editable in the batch scan review sheet.

[ Fixes ]
- Fixed the assistant dropping brand filters on grouped-count questions (e.g. "show me all Numakers with more than 2 spools").
- Fixed several double-tap and concurrent-edit bugs (onboarding sample data import, spool details, printer settings, inventory undo) that could create duplicates or let simultaneous edits silently overwrite each other.
- Improved reliability of cloud sync across multiple devices, fixing several cases where edits could conflict, go unacknowledged, or be lost during sync.
- Fixed desktop inventory keyboard shortcuts not working until the window was clicked first.
- Fixed bulk library trash/delete operations potentially leaving files in an inconsistent state if interrupted midway.
- Fixed batch UPC imports overwriting rather than updating existing barcode data.
- Improved performance when importing items by UPC.
- Fixed a crash that could occur when taking a photo in the scanner.
- Fixed an error when canceling the share sheet during CSV export on iOS.
- Fixed project print counts not updating correctly after deleting print logs.
- Fixed a possible crash when the Spoolman database cache failed to save on web.
- Fixed an issue with restoring purchases.
- Fixed a crash loading settings when a saved value had an unexpected type.
- Fixed temporary log files not being cleaned up if a log export failed.
- Fixed a crash when reordering favorites by dragging.
- Fixed scanning accepting invalid negative spool weights.
- Fixed very dark spool colors appearing pure black in light mode; true blacks are unaffected.
- Fixed barcode handling bugs in the batch scan review flow.

[ Security & Infrastructure ]
- Improved reliability of the web app's local database engine.
